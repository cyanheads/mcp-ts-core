# 0.9.2 — 2026-05-22

## Added

- **`multi-server-orchestration` skill** — workflow for parallel sub-agent fanouts across MCP server projects (greenfield builds, maintenance passes, release passes). Per-scenario references at `skills/multi-server-orchestration/references/`. Sub-agents spawned via the Agent tool don't inherit the parent session's skill registry; this skill mandates an orient block in every sub-agent prompt that runs the new `list-skills` script at step 4.
- **`scripts/list-skills.ts`** — parses YAML frontmatter from project-local `SKILL.md` files and prints an index. Wired into `package.json` `files:` and exposed as `bun run list-skills` in `templates/`.
- **`audit:refresh` npm script** — `rm -f bun.lock && bun install && bun audit`, added to root `package.json` and `templates/package.json`. Use when `bun audit` (via `devcheck`) flags a transitive advisory: Bun's `bun update` is sticky on transitive resolutions, so a stale-lockfile false positive can disguise an already-patched dep. If the advisory survives the refresh, it's real.

## Changed

- **HTTP landing page refactor** — pulls toward [#94](https://github.com/cyanheads/mcp-ts-core/issues/94) (broader redesign placeholder; not closing). Single ambient accent wash (was two radials), mono lowercase section heads with thin accent bar, card radius shrunk, padding tightened, mutability spine `3px → 2px`, no hover lift, no footer divider, description clamp `3 → 2` lines. Source links moved off the header — tool: actions row; resource/prompt: optional footer.
- **`fast-xml-parser` unpinned to `^5.8.0`** across `peerDependencies` and `devDependencies`; `resolutions` entry dropped; `devcheck.config.json` `outdated.allowlist` cleared.
