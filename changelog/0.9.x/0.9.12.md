# 0.9.12 — 2026-05-29

## Added

- **`GET /mcp` `server.keywords`** — consumer `package.json` `keywords` now surfaced on the status JSON alongside `name`, `version`, and `description`. Sourced via `resolveConsumerPackage()` in `src/config/index.ts`; threaded through `AppConfig.pkg.keywords` → `AppConfig.mcpServerKeywords` → `ManifestServer.keywords` → HTTP response. Omitted when empty, consistent with `homepage`. Override via `PACKAGE_KEYWORDS` env var (comma-separated). ([#165](https://github.com/cyanheads/mcp-ts-core/issues/165))

## Changed

- **`skills/orchestrations/SKILL.md`** v1.0 → v1.1 — sub-agent strategy expanded to three tiers (no fanout, parallel sub-agents, programmatic orchestration). Barriers narrowed to gate boundaries only. Rolling concurrency guidance defers to platform-native concurrency management. Orient block updated for `AGENTS.md`/`.codex` equivalents. Workflow-file creation guidance requires a back-pointer to `../SKILL.md`.
- **`skills/orchestrations/workflows/maintenance-release.md`** v1.0 → v1.1 — scope boundary note added: maintenance workflow does not consult, fix, or close issues; issue-driven work belongs to `fix-wrapup-release`.
- **Dependency bumps:**
  - `openai` ^6.39.0 → ^6.39.1

## Removed

- **`skills/migrate-mcp-ts-template/`** — workflow for converting mcp-ts-template forks to the package dependency model. Removed as that migration era is complete.
